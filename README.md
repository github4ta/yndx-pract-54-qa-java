# Инструкция

## Onboarding (для "новичков" проекта)
1. Для включения в список участников (collaborators) необходимо оставить заявку в пачке, в треде к посту [54_qa-java_theory](https://app.pachca.com/chats/30917502?message=716799825)
2. Склонируйте [репозиторий](https://github.com/github4ta/yndx-pract-54-qa-java.git)
3. Для выполнения задания (task) ВСЕГДА создавайте новую (рабочую) ветку (branch) от мастреа (master)
4. Не забывайте комитить (commits) в процессе написания кода отдельные (атомарные) этапы работы

Ресурсы
- Доска (board) с задачами (тасками, tasks) - [YNDX-Pract](https://app.clickup.com/90151597685/v/s/90158816179)

## Работа над задачами
- Выбираете для себя задачу с доски (Assign на себя)
- Обновляете локалную версию проекта (Git - master - Update)
- создаете рабочую ветку от мастера. Имя рабочей ветки = *ИДЕНТИФИКАТОР-ЗАДАНИЯ* (например, *CALC_DEV-0000*)
- пишете реализацию кода, комитите, пушите
- создаете Pull Request с именем *НАЗВАНИЕ-ВАШЕЙ-ТАСКИ* (например, *CALC_DEV-0010 – Калькулятор – метод(a, b)*)
- кидаете ссылку на ваш пулреквест в тред к посту о [командной работе](https://app.pachca.com/chats/30917502?thread_message_id=716799825&message=716799825)
- проходим этап код ревью
- разрешаем конфликты и мержим в мастер 

## Написание юнит-тестов
- Порядок работы анологичный работе над задачами (см. выше)
- Для именование тестовых методов предлагается придерживаться Test Case Naming Convention из **[статьи](https://www.baeldung.com/java-unit-testing-best-practices#3-test-case-naming-convention)**
